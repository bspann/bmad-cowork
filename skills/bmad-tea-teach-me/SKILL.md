# BMad TEA Teach Me (TEA Academy)

Use when learning about testing concepts, strategies, and best practices. Interactive learning companion with 7 structured sessions.

Trigger with: "teach me testing", "TEA academy", "learn testing", "testing tutorial"

## Instructions

**Agent:** TEA (Murat)
**Input:** User's current knowledge level and interests
**Output:** Interactive teaching sessions

### Step 1: Assess Level

Ask the user about their testing experience:
- Beginner: knows what tests are but hasn't written many
- Intermediate: writes unit tests, wants to improve coverage and strategy
- Advanced: experienced tester wanting to learn risk-based approaches and architecture

### Step 2: Present Curriculum

**Session 1: Testing as Engineering**
Why tests exist, the testing pyramid, cost of bugs at each stage.

**Session 2: Risk-Based Testing**
Probability × impact, P0-P3 prioritization, where to invest testing effort.

**Session 3: Test Design Patterns**
Given/When/Then, equivalence partitioning, boundary analysis, state transition.

**Session 4: Test Architecture**
Page objects, data factories, fixtures, test isolation, determinism.

**Session 5: Test Automation Strategy**
What to automate vs manual test, flaky test prevention, test maintenance.

**Session 6: CI/CD Testing**
Pipeline stages, parallelization, quality gates, artifact management.

**Session 7: Release Confidence**
Traceability, coverage metrics that matter, go/no-go decision frameworks.

### Step 3: Interactive Teaching

For each session:
1. Explain concepts with real-world examples
2. Ask comprehension questions
3. Provide hands-on exercises using the current project (if available)
4. Summarize key takeaways

### Step 4: Track Progress

The user can resume from any session. Ask which session to start or continue.
